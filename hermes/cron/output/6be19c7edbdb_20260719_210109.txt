**Top picks**
- **[HF discloses AI-driven intrusion — and used open weights to fight it](https://huggingface.co/blog/security-incident-july-2026)** — Hugging Face's July 16 incident writeup details an attack run by an "autonomous agent framework" that exploited a remote-code dataset loader and a template-injection in dataset config to run on a processing worker, then pivoted to cloud/cluster creds. The forensic work itself was blocked by commercial providers' safety guardrails, so HF used GLM 5.2 on its own infra to analyze 17,000+ attacker events. Concrete takeaway for builders: have a self-hostable model vetted *before* an incident, both to bypass guardrail lockouts and to keep attacker data/creds on-prem.
