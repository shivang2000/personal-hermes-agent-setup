**Top picks**

- **[Introducing Precursor: detecting agentic behavior with continuous client-side signals](https://blog.cloudflare.com/introducing-precursor/)** — Cloudflare extends Bot Management with a session-scoped JS signal collector that scores full user-journey behavior (wrist-pivot mouse arcs, cognitive-load click delays, tremor frequency) rather than single-shot challenges. Built as a complement to Turnstile; the explicit goal is making automation "more expensive to maintain and less reliable at scale." Worth understanding because it sets a likely default bar for what anti-bot at the AI-agent layer will look like in 2026.

- **[GhostLock (CVE-2026-43499): 15-year stack-UAF in the Linux rtmutex](https://nebusec.ai/research/ionstack-part-2/)** — `remove_waiter()` clears `current->pi_blocked_on` instead of `waiter->task->pi_blocked_on` on the `FUTEX_CMP_REQUEUE_PI` proxy path, leaving a dangling stack pointer whenever the PI walk reports deadlock. Only needs `CONFIG_FUTEX_PI=y`; no caps, no user namespaces. Affects v2.6.39 → v7.1-rc1; fixed in April 2026 (`3bfdc63936dd`). Google paid $92,337 via kernelCTF for a 97%-stable LPE + container-escape chain. Patch and `RANDOMIZE_KSTACK_OFFSET` mitigation notes included; LTS upgrades recommended.

- **[xAI's Grok Build CLI silently uploads entire repositories to a Google Cloud bucket](https://www.internationalcyberdigest.com/xais-grok-build-cli-uploads-entire-git-repositories-to-a-google-cloud-bucket/)** — Researcher `cereblab` mitmproxy-captured v0.2.93 shipping the full tracked repo + `.git` history + `.env` to `grok-code-session-traces` regardless of which files the agent actually opened; on a 12 GB repo, ~5.1 GB went to GCS vs ~192 KB of model traffic. A planted canary credential appeared unredacted. The "Improve the model" toggle governed training consent, not upload. xAI has since flipped a server-side `disable_codebase_upload: true` with no advisory; durability unverified. Treat as a reminder that any "local-first" agent CLI is only as isolated as its unseen egress.

- **[Terence Tao — Old and new apps, via modern coding agents](https://terrytao.wordpress.com/2026/07/11/old-and-new-apps-via-modern-coding-agents/)** — Tao ported ~24 of his 1999 Java teaching applets to JavaScript with an agent in hours, plus finally shipped a "Minkowski-Inkscape" relativity visualization he'd abandoned in 1999, plus a Gilbreath-conjecture visualizer for an active paper. One minor bug in 24 ports; agent found two bugs in the original code. Notable because this is a domain expert with zero incentive to oversell, and the failure mode (visualization aids, not proofs) is exactly the one most agents can credibly own today.

- **[Apple's new SpeechAnalyzer API, benchmarked against Whisper and its predecessor](https://get-inscribe.com/blog/apple-speech-api-benchmark.html)** — Inscribe ran Apple `SFSpeechRecognizer`, `SpeechAnalyzer`, and three Whisper sizes through identical code paths on LibriSpeech. `SpeechAnalyzer` cuts WER ~3.5–4× vs the legacy API (9.02% → 2.12% clean, 16.25% → 4.56% noisy) and beats Whisper-Small at ~⅓ the per-second compute. Methodology is unusually reproducible: raw transcripts + summary.json published, Whisper numbers reproduce OpenAI's within a small normalizer offset. The benchmark also surfaced a shipping bug in Inscribe's `finalizeAndFinishThroughEndOfInput()` handling. English-only, read-audiobook scope — caveats explicit.

**AI, Agents & Models**

- **[Claude Code sends 33k tokens before reading the prompt; OpenCode sends 7k](https://systima.ai/blog/claude-code-vs-opencode-token-overhead)** — Systima put a logging proxy between harnesses and the endpoint and found Claude Code's fixed system-prompt + tool-schema baseline is ~33k tokens vs OpenCode's ~7k; a 72 KB `CLAUDE.md` adds ~20k more, five MCP servers another 5–7k, and subagent fan-out multiplies the full baseline. Multi-step totals can flip either way depending on batching. Practical takeaway: cache accounting and tool-schema trim are now first-class perf work, not nice-to-haves.

- **[The real prices of frontier models: the tokenizer tax](https://playcode.io/blog/real-price-of-frontier-models)** — Playcode measured the same TypeScript file at 681 tokens on GPT-5.x vs 1,178 on Claude's newest tokenizer (+73%); Claude's own prior tokenizer was +31% leaner on the same code. The headline $/Mtok is the second of two multipliers on your invoice; the first (tokens per content) is invisible on every pricing page. Concrete numbers you can plug into agent-cost models.

- **[Control the ideas, not the code](https://antirez.com/news/169)** — antirez's latest: with current agents, "reviewing 5k lines of code a day" is the wrong control surface; LLMs are locally optimal but weaker on big ideas, so the leverage has moved to specifying, decomposing, and validating intent. He frames it as reassurance to less-seasoned programmers that the shift isn't "AI-pilled" weakness but a real change in where impact lives. Worth pairing with Tao above.

- **[I love LLMs, I hate hype](https://geohot.github.io//blog/jekyll/update/2026/07/12/i-love-llms.html)** — geohot's giddy-but-skeptical counterpoint: "the Year of the Linux Desktop is finally here" via opencode + local GLM, paired with a sharp rejection of both doomer narratives and "you missed the flash" narratives. Useful as a tone check on where credible practitioners actually are.

- **[GPT-5.6 Sol context cut: 353K → 258K despite advertised 1.05M — openai/codex #32806](https://github.com/openai/codex/issues/32806)** — A second regression inside a week; users report Codex running GPT-5.6-Sol is now seeing effective context windows of ~258K vs the documented 1.05M. Several "GPT-5.6-Sol stuck for hours" reports on X. Worth watching — if unaddressed, it materially affects long-context agent workflows.

- **[Migrating a production AI agent to GPT-5.6: 2.2× faster, 27% cheaper](https://ploy.ai/blog/migrating-a-production-ai-agent-to-gpt-5-6)** — Ploy's homepage-redesign eval: GPT-5.6 finished 2.2× faster, used ~half the output tokens, and produced more compact CSS than Opus 4.8. Roughly a third of first-pass "failures" were harness assumptions tuned to Opus (parallel tool calls, batched file reads, JSON-shaped output). The methodology section is more useful than the headline — a template for any cross-model migration.

- **[Show HN: Crucible — your AI wrote the tests, so who tested the tests?](https://github.com/Jott2121/crucible)** — A mutation-testing harness for AI-generated test suites: 71 real defects injected, a "Critic" agent writes more tests, and the loop refuses to bank a kill unless the new test fails on pristine code first. 332 tests, MIT, full run-time receipt per execution. Mutation scoring as a way to actually price test-suite quality, not just coverage.

**Cloud, Infra, Data & Security**

- **[Show HN: Clawk — disposable, network-restricted Linux VMs for AI coding agents](https://github.com/clawkwork/clawk)** — A third option beyond "babysit every prompt" and `--dangerously-skip-permissions`: `clawk` boots a Linux VM with the repo mounted in, an allow-list for outbound network, no host keychain, ssh-agent forward only for the allowed hosts. A pragmatic "the cloud provider can't see this layer of policy" answer.
