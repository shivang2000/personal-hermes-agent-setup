**Top picks**
- **[AsyncAPI supply-chain compromise via `pull_request_target` pwn request](https://www.wiz.io/blog/m-red-team-asyncapi-supply-chain-compromise-via-github-actions)** (Wiz) — 37 PRs hit `asyncapi/generator`; one exploited a workflow that checked out PR code in a `pull_request_target` job, exfiltrating a publish PAT and pushing 5 malicious `@asyncapi/*` npm versions (≈140K daily downloads). Payload is a new "M-RED-TEAM v6.4" stage-3 loader, not the same Shai-Hulud 2.0 actor. If you maintain any repo with `pull_request_target` + untrusted code checkout, audit it today.
- **[Cloudflare Precursor: continuous client-side bot scoring](https://blog.cloudflare.com/introducing-precursor/)** — A session-long behavioral JS shim complements Turnstile; it fingerprints human mouse dynamics (wrist-pivot arcs, physiological tremor, reaction latency) and feeds a real-time bot score to Cloudflare's Bot Management. Raises the cost of "session-shaped" automation — important if you protect login, signup, or pricing endpoints with Cloudflare.
- **[Claude Code uses ~33K tokens of harness payload per turn vs OpenCode's ~7K](https://systima.ai/blog/claude-code-vs-opencode-token-overhead)** (Systima) — Logging-proxy measurement on the same model/machine: Claude Code also rewrites cache prefixes every turn (up to 54× more cache-write tokens), and a 72KB `AGENTS.md` + 5 MCP servers pushes a "real" baseline to 75–85K tokens before the user types. Concrete reason to slim instruction files and audit MCP bloat if your CC bill is climbing.
- **[Ploy migrated a production marketing-website agent from Claude Opus 4.8 → GPT-5.6 Sol](https://ploy.ai/blog/migrating-a-production-ai-agent-to-gpt-5-6)** — 2.2× faster wall-clock, 27% cheaper, ~½ the output tokens, equal or higher visual score — but only after they fixed their eval harness. ~⅓ of initial "failures" were harness assumptions (sequential vs parallel tool calls, missing `minScore` defaults). If you eval frontier models, triage traces before trusting pass rates.
- **[OpenAI Codex `MultiAgentV2` now encrypts inter-agent messages — audit trail lost](https://github.com/openai/codex/issues/28058)** (openai/codex#28058) — PR #26210 marks `InterAgentCommunication.message` as encrypted, leaving only `encrypted_content`, so local rollouts/traces no longer show what `spawn_agent`/`send_message`/`followup_task` actually sent. Real privacy win, real observability loss for anyone debugging sub-agent runs.
