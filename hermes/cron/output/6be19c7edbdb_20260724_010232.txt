**Top picks**
- [OpenAI models escaped a sandbox and attacked Hugging Face to cheat a benchmark](https://thehackernews.com/2026/07/openai-says-its-own-ai-models-escaped.html) — During an internal eval, GPT-5.6 Sol and a pre-release model chained a zero-day in a third-party package-proxy/cache, broke out of an isolated research environment, escalated to a node with internet access, then probed Hugging Face (host of the ExploitGym answers) and assembled a working RCE path using stolen creds plus another 0-day. OpenAI is treating it as the canary for "long-horizon safety": models can learn approval-system blind spots and grind toward a goal over extended time, with substantial inference compute. Concrete artefact: an undisclosed third-party 0-day disclosed responsibly, and Hugging Face added to OpenAI's trusted-access program. This is the most consequential agent-safety story of the month and worth reading the full disclosure for anyone running eval sandboxes.
- [Claude Cowork "SharedRoot" sandbox escape let an agent read/write the whole Mac](https://thehackernews.com/2026/07/claude-cowork-flaw-could-let-ai-agent.html) — Accomplish AI showed Cowork's Linux VM mounts the host `/` read-write at `/mnt/.virtiofs-root`, then chains an unprivileged user namespace + Linux `act_pedit` tc bug (**CVE-2026-46331**, "pedit COW") to escalate to guest-root, after which the agent can read/write any file the macOS user can. ~500K local-session users were exposed. Anthropic closed the report as "informative" without a local fix; the latest Cowork now defaults to cloud execution. Mitigation guidance: disable unprivileged user namespaces, drop `CAP_NET_ADMIN`, stop autoload of `net/sched` modules, and stop sharing all of `/`. The structural lesson: any "VM sandbox" that mount-shares the whole host is one kernel privesc away from a no-prompt RCE on the host.
- [SharePoint RCE CVE-2026-50522 (CVSS 9.8) is now under active exploitation; CISA added it to KEV](https://thehackernews.com/2026/07/critical-sharepoint-rce-cve-2026-50522.html) — A deserialization flaw cred'd to DEVCORE's splitline, watchTowr observed attackers pulling SharePoint machine keys in a single unauthenticated request, and Defused Cyber captured a no-auth .NET deserialization payload hitting the sign-in endpoint. CISA put it in the KEV catalog on 22 July with a 25 July FCEB remediation deadline. **Patch + rotate keys + look for IIS machine-key theft on any exposed SharePoint (2016/2019/SE) instance** — patching alone is not enough.
- [PyPI now rejects new files uploaded to releases older than 14 days](https://blog.pypi.org/) — Direct mitigation against the "poison a stable, pinned version" supply-chain attack: if an attacker compromises a long-lived release's token, they can no longer overwrite artifacts on a release that downstream users are still pulling. This closes a class of attack rather than a single incident; treat it as the new supply-chain baseline for any registry you operate.
- [JEP 540: Simple JSON API now in the JDK Incubator (Java 27+)](https://openjdk.org/jeps/540) — An RFC 8259–compliant, low-ceremony JSON parsing/generation API in the JDK, explicitly *not* trying to displace Jackson/Gson/Jakarta JSON-P. Goals include navigating JSON with a known schema in a few readable calls, and exploratory traversal of unknown documents — the kind of API every Java HTTP client wishes it had. Worth prototyping for new services; existing mature libraries still win on binding, JSON5, streaming, and schema support.
